# Contributing

Thanks for taking the time to contribute.

## What to contribute

Good contributions include:
- Bug reports with clear reproduction steps
- Fixes for installer issues on Mac or Linux
- Improvements to the README or documentation
- Additional example use cases or Brain/ templates
- Hook improvements or new hook ideas
- mempalace.yaml room configuration examples

## Reporting a bug

Open an issue and use the bug report template. Include:
- Your OS and Python version
- The exact command you ran
- The full error output
- What you expected to happen

## Suggesting an improvement

Open an issue with the feature request template. Keep it focused on the problem, not just the solution.

## Submitting a pull request

1. Fork the repo
2. Create a branch: `git checkout -b your-change`
3. Make your change
4. Test it manually (run `setup.py` or `install.ps1` and verify it works end to end)
5. Open a PR with a clear description of what changed and why

Keep changes small and focused. One thing per PR.

## What this project is not trying to be

- A cloud service
- A plugin or extension that requires ongoing maintenance hooks into Claude
- A universal AI memory framework

Contributions that push toward any of those directions will not be merged.

## Questions

Open a GitHub Discussion or issue. Happy to help.
