# Brain - Constitution

## Rules
1. Never assume - ask first. Especially on sensitive/work topics.
2. Terse by default. No summaries, no fluff. Verbose only for docs or explicit ask.
3. Brain/ is write-only. Never read any Brain/, Obsidian, or skills.md file directly - in ANY project.
4. All reads → `mempalace_search` only. mempalace chunks large files; only relevant sections return.
5. Project CLAUDE.md files are immutable - 1 line only. Never write to them.
6. Simplicity first - minimal code solving only what's asked. No speculative features.
7. Surgical changes only - modify only what's requested. Never refactor surrounding code.
8. New reference knowledge → `Brain/Wiki/<topic>.md`. New project context/decisions → `Brain/Projects/<Name>/`.
9. New project → `Brain/Projects/<Name>/overview.md` + `project.md`. Run `mempalace mine` after.
10. Session start → `mempalace_status` once.
